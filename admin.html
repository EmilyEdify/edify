This tutorial goes through the workflow for creating a new blog post and touches on how to find and add blog post content.

Learn about the important elements that make up a Blog Post

There are three parts to that make up a blog post and each is described below.
It's important to assemble these correctly as they will enable the code rendering the files to do so properly.

Part 1: The Content

Files:

All blog post files live in the folder `_posts`.

There are a few key things to know about the blog post files:

  • The file ending is `.md` (short for markdown);
    this enables you to use a shorthand type of html that's faster and easier to type (more on this in a moment).
  • The file naming pattern is `YYYY-MM-DD-blog-post-name.md`;
    this is important in Jekyll so it lists and renders post chronologically (newest on top). The url for each post will only show `blog/blog-post-name`.
  • An important thing about file naming: files names should have NO capital letters, NO spaces, and NO special characters
    Characters like: ?, /, &, $, will cause problems in any file name. However, dashes are encouraged. for example: `2017-05-25-blog-post-name.md`
Images:

Images are not necessary for a post to work, to make a post without an image, simply leave the image meta data blank.

However, if you do have an image, follow these guidelines for best results:

  • Image file names should have NO capital letters, NO spaces, and NO special characters
    Characters like: ?, /, &, $, will cause problems in any file name. However, dashes are encouraged. for example: `image-file-name.jpg`
  • Image file formats try to stick with `.png` and `.jpg` file formats - some of the others can cause problems and these two are more than enough to get the job done!
  • Image sizes: Images are rendered three different ways in the blog; on the post listing, in the sidebar, and in the body of the post itself. The image size defined here is intended to maximize load-time efficiency & accomodate layout proportions without compromising the image quality.
    • Image should be at least 4-5 inches tall in a landscape ratio (wider than tall) - css layout should adjust to accommodate some variation within these parameters.
    • Image size assumes the picture resolution is 72 (common web image resolution - as opposed to print resolution which is 300)
  • Multiple images: currently, blog post pages are not optimized for more than one image per post. While it is totally possible to add multiple images, they may not align perfectly. (Though they may align just fine - it depends on the rest of the content.) *This is current as of 6/9/2017
  • Image file location: all blog and client images are saved in the folder named "img". This is set to default in Prose.io so you shouldn't need to do anything with this.

Part 2: The Meta Data

Meta data (aka "FrontMatter" in Jekyll) is the list of variables that allow you to enter the information once and it can be duplicated all over the site - saving you a lot of time and effort.

Prose has some very handy config options that make editing this super easy (watch the video to see the workflow) but here's a breakdown and description of each part of the meta data you will find in prose.

You open the metadata panel by clicking the metadata icon in the upper right corner

  1. Publish Date:

    This is here so you can update a post (like the services post that the process cards link to) so the date will look current without having to rename a file.

  2. Add Category:

    This is set up using the tag feature in order to let you add your own categories by typing the name into the text field OR you can select an existing category from the dropdown menu. HOWEVER, the blog isn't set up to accept more than one category per post (we can create tags to do this if you want) so if you do accidentally add more than one category, the site will ignore all but the first category listed.

    One other cool thing - you can just type the format you want to be displayed - no need to mess around with "alternative_learning", you can now use "Alternative Learning". Or whatever you want.

  3. Feature Post:

    Check this option to have a post be automatically included on the home page. It is designed to accommodate three posts. Fewer posts may look unbalanced but you may be able to set six to show up there if you wish.

  4. Post Width:

    This will determine the size of the image in the main post body - the default is "half-width". If you have a post where you want a full width image, I recommend making sure the image is on the larger size of the image size guidelines (above) and that the image quality is reasonably high.

  5. Image File Name:

    This is where you type the file name of the image you will download using the prose.io download feature. This allows the image to appear in all the other locations blog images appear that are not the body of the post itself.

  6. Image Alt Text:

    This is for Universal Design purposes - write a brief but descriptive bit about the image here.

  7. Short Description:

    This is the text that will appear on the blog page and on the category list pages.

  8. Raw Metadata:

    This is just a default of prose.io - you can just ignore it.

All client text content goes into the file 'clients.yml' file and client logos get uploaded in the same way as blog posts.

Below are the data items you can include in a new client listing. Don't worry if you don't have all these pieces! You can leave a section blank by simply leaving empty quotes, like this => "".

  1. 1. Client Company Name
  2. 2. Project End Date
  3. 3. Client Website URL
  4. 4. Client Logo Image File
  5. 5. Client Name (The person who gives the testimonial)
  6. 6. Client Job Title
  7. 7. Testimonial Text
Guidelines for Client Logo Image Files:
  • Image file names should have NO capital letters, NO spaces, and NO special characters
    Characters like: ?, /, &, $, will cause problems in any file name. However, dashes are encouraged. for example: `image-file-name.jpg`
  • Image file formats must be a `.png` file with a transparent background.
  • Image size requirements = 3 inches wide x 1 inch tall (set largest dimension to the limit and then enlarge the canvas size to meet size requirements)

Step 2: Copy the Template

Click the button below to copy the "New Client" template.
Important: the alignment of this is critical! The " - " must be as far to the left as possible and each subsequent line must match alignment indicated below.

  project_end_date: "Insert project end date here"
  website: "Insert client website here"

  logo: "insert client logo file name here"

  client_testimonial_name: "Insert client name here"
  client_testimonial_job_title: "Insert client job title here"

  testimonial: |
    "Insert testimonial text here"

Click the link to open the "New Client" page.

Then paste the template at the bottom of the file and edit all the text fields.

Leave the fields you don't have content for empty like this -> ""

Watch the tutorial on editing Flip Cards

This tutorial goes through the workflow for adding a new flip card or editing content for an existing flip card.

About Flip Cards:

All client text content goes into the file 'questions.yml' file.
It's important to get the syntax correct so the code renders the text correctly.

The flip cards are edited in the same way as the client data but with just two data points; the front side questions, and the back side answers.

Go directly to the editing page

Things to know about the Flip Cards:

  • Number of Flip Cards: You can have as many or as few cards as you want. The CSS is set up to render the files in rows of three, so the ideal number of cards is either 3, or 6. However, it will accommodate any number you wish.
  • Number of characters per card: The styling is set up to accommodate a maximum number of characters without making the text flow off the cards. Character count includes spaces and special characters like " , . - ? etc.
    • FRONT "question side": The maximum number of characters for the front side is 145 (though I recommend keeping the total number at or below 140 total characters)
    • BACK "answer side": The maximum number of characters for the back side is 190 (though I recommend keeping the total number at or below 185 total characters)
  • The "Learn More" Button: The button on the back links to the blog post "services" which can be edited here to reflect the most current information about Edify services.
    This can be adjusted easily to go to the contact page, to a different page, or even to individual blog posts for each question/answer combo. If you decide you want to change this, contact me.

Watch the tutorial on adding a new services page

This tutorial goes through the workflow for adding a new services page.

There are 4 steps to adding a New Services Page:

Step 1: Create a New Page

  • 1. Go to the Edify Prose.io "root directory". This means you aren't in a folder and there isn't any bigger "container" you can click into. It should look something like the image to the right.
  • 2. Click the green "New File" button. You will be taken to a new file page. Give the new services file a name - this should follow a somewhat simple and direct format - ideally it will begin with the word "services".

    For example:
    services-1.md
     services-2.md
     services-3.md
     or
    services-onboarding.md
     services-managers.md
     services-knowledge.md

    The simpler and more consistent the file name, the easier it will be to find, update, and edit later on.

  • 3. Save the new file with a commit message the same way you do for any other save in Prose.

Step 2: Add "Meta Data"

At this point, your new services file will not have the "meta data" icon on the right. The Meta Data is critical for the page to render properly and link to the flip cards the way you want.

To get this working correctly, you will need to add the Meta Data to the top of the body of the new file and then save it.

To Add Meta Data:
  • 1. Click the Copy Button below to put the Services Meta Data template into your paste buffer.
  • 2. Paste the template into the very top of the body of your new file.
  • 3. Make absolutely certain the aligment is correct - in this case, each new line (layout, title, meta_title, meta_tag, and published) are all the way to the left. The line above "layout" should be three dashes like this: '---' and the line after "published" should also have three dashes '---'.
  • 4. Now save this with a commit message.
    Important Note: Remember your file name - although it will display as the name you gave it when you created it (so you can look in the root directory to check if you forget), once you add the Meta Data, the name of the file will appear as the title in the template you pasted in.

Once you do this, the Meta Data icon should appear on the right. If you do not see it, try waiting about 30 seconds and then pressing "Command + Shift + R" or on a PC; "Control + Shift + R" to force the browser to display your most recent changes. 

---
layout: services-page-layout
title: Services Edify Offers

meta_title: Edify Learning Design Process is data-driven & human-focused
meta_tag: >-
  At Edify, we believe that great learning design means; designing for people
  first, practicing empathy, anticipating needs, honoring diverse approaches,
  and providing scaffolding and offer accessible options.
published: true
---

Step 3: Add Page Content

Now you can update your Meta Data. To do this, click the Meta Data icon on the right - it's the one with several horizontal lines.

You can give the file a new title and a new meta_title and meta_tag. Click "Done" when you are finished.

For more info about the Meta Title and Meta Tag, check out the tutorial info in the "Google Analytics and SEO" tab on this page.

Important Note: Do Not change the top line labeled "layout" - changing this will cause the page to fail to render.

Step 4: Update Flip Card Button Link URL

Finally, you will need to change the URL for the corresponding flip card on the process page so that when people click the button on the back, they will be directed to the new services page.

To do this, open the questions.yml file in the _data folder and find the question you want directed to the new services page.

Change the "button_link" text to be the name of the new file.

Important Note: Make sure you change the file name to the actual file name - not the title that shows up after you add the Meta Data. This is the file name that (ideally) begins with the word "services"

What are meta titles?

"A title tag is an HTML element that specifies the title of a web page. Title tags are displayed on search engine results pages (SERPs) as the clickable headline for a given result, and are important for usability, SEO, and social sharing. The title tag of a web page is meant to be an accurate and concise description of a page's content."
~ Moz SEO Tutorial on Title Tags

What are meta descriptions?

"Meta descriptions are HTML attributes that provide concise summaries of webpages. They commonly appear underneath the blue clickable links in a search engine results page (SERP)."
~ Moz SEO Tutorial on Meta descriptions
